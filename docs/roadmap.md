## P1 · 下一步（高优先级）

### 1.1 浴室真实环境数据收集
**为什么**：识别率 / 误触发率 / 省电表现只有真实场景才测得准，自动化测试覆盖不到。
**怎么做**：洗澡时用一段时间，手动记录几次洗澡的：
- 命中次数 / 误识次数
- 30 分钟后服务是否被杀
- 平均响应延迟（主观感觉）
**完成定义**：把数据追加到 `docs/verify/` 目录（自建），持续 ≥ 3 次洗澡。

## P2 · 应该做（中优先级，取决于 P1 数据）

### 2.1 App 内自启动白名单引导
**为什么**：OriginOS 后台杀进程是最大体验杀手；现在靠用户自己去系统设置。
**怎么做**：首次启动在 onboarding 里加"为获得稳定体验，请到系统设置开启自启动"的一键跳转按钮。检测不同国产 ROM（Vivo / OPPO / 小米 / 华为）对应不同设置页的 intent。
**完成定义**：Vivo X200 Pro 上一键跳转到本 App 的自启动开关页面。
**Blocker**：需要实测 OriginOS 的具体 intent action。

### 2.3 手势坐标调试面板
**为什么**：抖音版本更新可能让默认坐标失效；不同分辨率手机可能需要微调。
**怎么做**：MainActivity 加一个隐藏入口（长按标题 3 秒），打开调试页面，四个滑块调 `centerXPct` / `swipeTopYPct` / `swipeBottomYPct` / `tapYPct`，存到 DataStore，`GestureConfig.default()` 改成从 DataStore 读。
**完成定义**：能在不重新编译的情况下调整坐标并生效。
**预估**：~100 行代码，半天。
**前置条件**：✅ DataStore 配置持久化已完成（2026-04-29）

## P3 · 可以做（低优先级，按需）

### 3.1 快手 / 视频号 / 小红书适配
**为什么**：扩展可用场景。
**怎么做**：使用 `SupportedApp` 架构添加新 App，实测手势坐标是否需要专属配置。
**风险**：每个 App 的 feed 结构不同，手势坐标可能不通用。
**前置条件**：✅ 多 App 支持架构已完成（2026-04-29）

### 3.2 更多语音命令
**候选**：点赞、收藏、关注、音量 +/-、回到开头。
**原则**：只加实际场景会高频用到的。需要先用 MVP 用一段时间，统计"我最想说但说不了"的命令是什么。

### 3.3 Vosk 仪器测试实机跑一遍
**背景**：Task 8 的 `VoskRecognizerTest.kt` 代码已写但因 Vivo USB 安装权限拦截未执行。
**怎么做**：可以换一种运行方式（Android Studio 里点"运行"，给够时间点"允许"）。
**完成定义**：`./gradlew :app:connectedDebugAndroidTest` 绿色。

### 3.5 颜色 / 主题跟随系统深色模式
**现状**：写死了 iOS 蓝。
**怎么做**：用 `MaterialTheme.colorScheme` 的语义色代替硬编码颜色，或者提供浅/深两套自定义色板。

## 待规划 · 没想清楚的

- 唤醒词（"小隔 下一条"）防误触——需要先收集真实误触数据判断是否必要
- 云端识别作为离线识别失败时的 fallback——值不值得额外依赖、额外的隐私成本？
- 多用户场景（家里有共用手机）——不是本人的声音是否也触发？是 feature 还是 bug？

## Archived · 放弃或已过时

### 提示音反馈（SoundPool → ToneGenerator → 已移除）
**原因**：SoundPool 加载异步，触发时常"not READY"；ToneGenerator 在浴室水声下仍不明显；用户明确要求改视觉反馈。
**决策日期**：2026-04-18
**替代方案**：`TYPE_ACCESSIBILITY_OVERLAY` 顶部横幅（见 spec §4.5）

### 严格关键词匹配
**原因**：Vosk 中文小模型对"下一条"等复合词切分不稳定，会漏字（"下条"）或多字（"[unk] 下一"）。
**决策日期**：2026-04-18
**替代方案**：模糊匹配 + `[unk]` 过滤（见 spec §12.6）
